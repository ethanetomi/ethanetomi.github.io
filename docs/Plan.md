# Project Plan

## Project Overview

This project will create a professional two-page website for Sentinel Spectrum Technologies, a fictional defense contracting company specializing in software-defined radio (SDR) solutions. The project will be developed using HTML, CSS, Git, GitHub, and GitHub Pages while following the project management principles covered throughout this course.

---

# Risk Analysis (TAME)

## T - Threats

- Falling behind schedule because of military obligations and limited free time.
- Introducing HTML or CSS errors while making changes.
- Missing required project documentation.
- Publishing an incorrect or broken GitHub Pages link before submission.

## A - Assumptions

- GitHub Pages will remain available throughout the project.
- My computer, internet connection, and software will function properly.
- The project requirements will not significantly change before the deadline.
- Previous assignments can be reused as the foundation for this project.

## M - Mitigations

- Complete the project in stages instead of waiting until the deadline.
- Test every webpage after making changes.
- Commit changes to GitHub frequently to create recovery points.
- Review the grading rubric before the final submission.
- Verify that all required Markdown documents are included in the repository.

## E - Exceptions

- Unexpected internet outages.
- Computer hardware or software failures.
- Emergency military obligations.
- Changes requested by the instructor after project completion.

# Work Breakdown Structure (WBS)

## Phase 1 - Planning

- Complete Project Scope Statement
- Review assignment requirements
- Organize repository structure

## Phase 2 - Content Development

- Write homepage content
- Write About page content
- Create company mission and services

## Phase 3 - Website Development

- Build index.html
- Build about.html
- Create style.css
- Add navigation between pages
- Apply responsive styling

## Phase 4 - Testing

- Test all navigation links
- Verify formatting
- Test GitHub Pages deployment
- Correct HTML and CSS issues

## Phase 5 - Documentation

- Complete README.md
- Complete plan.md
- Complete retrospective.md
- Review repository organization

## Phase 6 - Final Submission

- Publish GitHub Pages website
- Perform final quality review
- Submit GitHub repository link in Canvas

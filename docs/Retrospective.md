# Project Retrospective

## What Went Well

One of the biggest successes during this project was becoming more comfortable using GitHub and organizing a project from beginning to end. I also improved my HTML and CSS skills while creating a website that looks more professional than anything I had built before. Planning the project before writing code made development much smoother and helped me stay focused on the project requirements.

---

## What Went Wrong

The biggest challenge throughout this course was balancing college assignments with Army AIT. There were weeks where I had very limited time to work on assignments, which caused me to fall behind at times. I also learned that waiting too long to start larger projects creates unnecessary stress, even if the technical work itself is not difficult.

---

## What I Would Do Differently

For my next project, I will begin planning much earlier and divide the work into smaller milestones instead of trying to complete large portions at once. I also plan to make more frequent Git commits so every stage of the project is documented and easier to recover if something goes wrong. Finally, I will continue using project management techniques such as defining the scope, identifying risks, and reviewing completed work before submission because they make the entire development process much more organized.
